常见的三大请求API

2025.03.05

Xhr

创建实例 let xhr = new XMLHttpRequest()

打开请求 xhr.open([http method], [url], [isAsync])

发送请求 xhr.send()

事件监听 readystatechange 状态码发生改变

readyState 属性

0： 刚创建实例

1： 已调用 send

2： 接收到全部响应内容

3： 正在解析响应内容

4： 响应内容解析完成

Axios（第三方）

axios 是使用 promise 对 xhr 的一层封装。

  1. axios.get/delete(url, [config])
  1. axios.post/put/patch(url, [data], [config])

Fetch（ES8 api）

请求返回4xx 5xx 状态码时不会reject，只有网络出错导致请求不能完成时reject

Content-type 数据
text/plain 纯文本数据
application/x-www-form-urlencoded 键值对
multipart/form-data 二进制流（传输文件）
application/json JSON

RESTful API 的 URL 设计规范

  • 所有单词使用英文小写，用复数表示
  • 连字符用中划线 ‘-‘，URL结尾不带有 ‘/‘
  • URL即资源，其中不应出现动词，用请求方式表示动作

HTTP Method

GET 请求用来获取资源

POST 请求用来新增资源

PUT PATCH 请求用来更新资源

patch 相对于 put 是局部更新一个资源。

比如修改一个用户的昵称：
put 会传递整个 userInfo 对象，
而 patch 只传部分字段 nickname

DELETE 请求用来删除资源

HTTP请求的安全性和幂等性

幂等： 在计算机范畴内表示一个操作执行任意次对系统的影响跟一次是相同
